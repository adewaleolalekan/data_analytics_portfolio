Project Overview

This project demonstrates a complete, production-style web scraping pipeline built with Python. Given a business type and location, the scraper navigates Google Maps, scrolls through the results feed to collect listings, then clicks into each one to extract detailed business information.

The resulting dataset is deduplicated by phone number and business name, then exported to both CSV and a formatted Excel (.xlsx) file — ready for CRM import, market research, or further analysis.

How It Works

The scraper follows a five-stage pipeline from search to clean export.

01
Navigate & Search

Launches a headless Chromium browser via Playwright with a realistic user agent and viewport. Navigates to a Google Maps search URL constructed from the query and location arguments.

02
Scroll & Collect Listings

Programmatically scrolls the results sidebar, counting unique listing URLs after each scroll. Stops when the requested limit is reached or no new results appear after five consecutive scrolls.

03
Extract Detail for Each Listing

Clicks into each unique listing and extracts: business name, category, star rating, review count, address, phone number, and website URL — using precise CSS selectors and aria-label targeting.

04
Deduplicate & Validate

Filters out records without phone numbers, then deduplicates by phone and business name using pandas — ensuring every exported row is a unique, contactable business.

05
Export to CSV & Excel

Saves two timestamped output files: a flat CSV for programmatic use, and a formatted Excel file with auto-sized columns via openpyxl — both ready for CRM import or further analysis.

Key Features

⚡ Async Architecture

Built with Python's asyncio and Playwright's async API for non-blocking browser control.

🌐 Dynamic Content Handling

Headless Chromium renders JavaScript-heavy pages that traditional HTTP scrapers cannot access.

🧹 Built-in Deduplication

Deduplicates by phone number then by name, so the exported dataset has no repeated businesses.

📊 Dual Export

Outputs to both CSV and a formatted .xlsx file with auto-sized columns — no post-processing needed.

🖥️ Live Progress Display

Real-time terminal feedback via the Rich library — spinner, progress count, and a results summary table.

🛡️ Graceful Error Handling

Per-listing try/except blocks mean a single failed extraction doesn't abort the entire run.
