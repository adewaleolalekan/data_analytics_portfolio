Data Cleaning Case Study

Preparing a student performance dataset for analysis using sorting and filtering techniques in spreadsheets.

Project Scenario

As a data analyst for a large public school district in Portugal, I was tasked with preparing a dataset for an upcoming analysis. The superintendent wanted to understand the factors affecting student grades in two high schools: Gabriel Pereira (GP) and Mouzinho da Silveira (MS).

The Challenge

The raw dataset, collected from academic reports and student surveys, contained student grades, background information, and study habits. However, before any analysis could be performed, it was critical to clean the data. An analysis based on "dirty" data—data with incorrect, missing, or faulty values—could lead to flawed conclusions and ineffective policy changes.

My Data Cleaning Process

1

Initial Exploration with Sorting

My first step was to get a feel for the data. I performed a multi-level sort on the entire dataset: first by `school` (A-Z), and then by `age` (Z-A). This immediately organized the data and revealed a potential issue: the age range for students in both high schools appeared to be 15-22. Since the dataset was for high school students, ages over 19 were highly suspicious and likely data entry errors.

2

Removing Invalid Data with Filtering

After confirming with the superintendent that the valid age range was 15-19, I needed to remove the incorrect entries. I applied a filter to the `age` column and selected only the invalid ages (20, 21, and 22). This isolated the nine rows containing incorrect data, which I then deleted from the spreadsheet, ensuring the age data was now accurate.

3

Handling Missing Data

The `reason` column, which explained why a student chose their school, was important for the analysis but contained several blank cells. To handle this, I filtered the `reason` column to show only the blanks. Since I couldn't know the actual reason for each student, I filled these missing values with a consistent placeholder, `none_given`, using the fill handle feature to apply the change to all blank cells efficiently.

4

Transforming Data for Analysis

The final cleaning task involved the `Medu` (mother's education) and `Fedu` (father's education) columns. The data was in text format (e.g., "higher education"), which isn't suitable for many statistical calculations. To prepare for analysis, I converted this text data into numerical codes (e.g., "higher education" became `4`). I used the "Find and Replace" function to systematically replace each education level with its corresponding numeric code across both columns.

Project Outcome

Through this methodical process of sorting, filtering, and transforming the data, I successfully cleaned the student performance dataset. The result was a reliable, accurate, and properly formatted dataset, ready for the analysis phase. This foundational work is critical to ensure that the insights derived for the school district are valid and can lead to effective, data-driven decisions.
